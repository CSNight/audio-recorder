# 2026-06-23 开发日志

## 完成事项

### Phase 5 第一阶段：G.711 编码器

完成了 G.711 A-law / U-law 编码器的全部实现，对应 phase5 计划中「第一阶段：G.711（1-2 天）」。

**实现文件**

```
src/codecs/g711/
  ├── index.ts                 Vite 入口，注入 Worker factory，导出两种编码器定义
  ├── g711-encoder.ts          A-law / U-law 算法（TypeScript 重写）
  ├── g711-chunked-encoder.ts  流式分块编码器（ChunkedEncoderDefinition）
  ├── g711-snapshot-exporter.ts 全量快照导出（SnapshotEncoderDefinition）
  ├── g711-worker.ts           Worker 入口
  └── types.ts                 G711Variant、G711ExportOptions、G711ExportResult
```

**关键决策**

- 选择 TypeScript 重写算法，而非沿用 `vendor/g711x.js`。G.711 核心是查表 + 位运算，算法极简，重写成本低，可获得类型安全且无 vendor 文件依赖。
- 多声道输入只取 `planar[0]`（G.711 是单声道电话标准，不支持多声道）。
- `g711-snapshot-exporter` 通过 `resample()` 支持目标采样率（通常 8000 Hz），重采样在导出时按需发生，不影响录音采集。
- Worker 复用 `createWorkerMessageHandler` 工厂，与 mp3 Worker 结构一致。

**算法说明**

- A-law（PCMA）：ITU-T G.711 欧洲/国际标准，13 段折线量化，输出 XOR 0xD5
- U-law（μ-law / PCMU）：北美/日本标准，15 段折线量化，输出 XOR 0x7F / 0xFF
- 输入：16-bit signed PCM；输出：8-bit 压缩字节（0–255）
- 段号定位使用 64 项查找表（`pcm_val >> 8 & 0x7F`），与原始参考实现一致

**测试**

完成编码算法正确性验证，A-law / U-law 输出与参考实现（vendor/g711x.js）比对通过。
