## PR1 — API 展平 + 命名重构

- 目标：消除 `capture` 嵌套层，移除公开 API 中的内部概念，统一命名语义。
- 完成：
  - `CreateRecorderOptions` 展平：输入参数从 `input?: RecorderInputOptions` 改为直接继承 `RecorderInputOptions`，`storage` 并列放置。`createRecorder()` 内部通过 `{ storage, ...inputOptions } = options` 分离。
  - `RecorderOpenOptions = RecorderInputOptions`（类型别名而非空接口），避免 ESLint `no-empty-object-type` 报错。
  - `sourceStream` 从公开类型降为内部字段，在 `recorder-controller.ts` 的 `open()` 中通过类型断言提取，不在公开签名暴露。
  - `src/capture/` 目录整体迁移为 `src/input/`，所有 `Capture*` 类型改为 `Input*`。
- 踩坑：
  - `exactOptionalPropertyTypes: true` 严格模式下，`sourceStream?: MediaStream` 不能赋值给要求 `| undefined` 的字段，需显式写成 `sourceStream?: MediaStream | undefined`。
  - 删除 `src/capture/` 后出现 `CaptureSessionState` 引用报错，确认无外部依赖后直接删除。
- 变更文件：
  - `src/types.ts`
  - `src/input/types.ts`（原 `src/capture/types.ts`）
  - `src/input/browser-input-adapter.ts`（原 `src/capture/browser-capture-adapter.ts`）
  - `src/input/browser-input-session.ts`（原 `src/capture/browser-capture-session.ts`）
  - `src/input/input-graph.ts`（原 `src/capture/capture-graph.ts`）
  - `src/core/recorder-controller.ts`
  - `src/index.ts`

## PR2 — 移动端帧缓冲 + 丢帧补偿

- 目标：移动端 AudioWorklet 帧频过高（375Hz）导致积压，vendor 算法补偿丢帧。
- 完成：
  - 帧缓冲只在移动端 + AudioWorklet 路径启用，桌面端和 ScriptProcessor 路径不变。`batchSamples` 通过 `processorOptions` 传入 worklet，而非注入 module source 字符串（保持 blob 内容稳定）。
  - 丢帧补偿算法精确还原 vendor `envInFixTs`：滑动 3 秒窗口、双重门槛（lost > tsIn/3 且冷启动保护通过）、缺口超过本帧 1/5 才补偿。`resume()` 时重置窗口，避免暂停时间被误判为丢帧。
  - `frameLossCompensation?: boolean` 对外暴露开关（默认 false = 开启补偿），内部字段名 `disableEnvInFix` 与 vendor 对齐。检测和 warning 上报无论开关都执行。
  - `processFrame()` 抽为私有方法，静音帧和正常帧共用同一处理路径。
- 踩坑：
  - vendor 用 `Date.now()`，本库用 `performance.now()`，精度更高，行为一致。
  - `batchSamples` 参数在 TypeScript 中声明但 worklet 代码字符串内不直接使用，改名为 `_batchSamples` 消除 unused 警告。
- 测试结果：
  - `browser-input-session.spec.ts` 新增 4 个用例，`input-graph.spec.ts` 新增 3 个用例。
- 变更文件：
  - `src/input/input-graph.ts`
  - `src/input/browser-input-session.ts`
  - `src/types.ts`

## PR3 — 能力预检 + 插件事件命名规范化

- 目标：`open()` 前同步感知采集链路，插件事件与核心事件命名空间隔离。
- 完成：
  - `checkRecorderCapability()` 纯同步无副作用，只检测构造函数/静态方法存在性，不实例化对象，不请求权限。可在页面加载时就调用。
  - `open()` 内预检 + 提前上报 warning，将 ScriptProcessor 降级的感知时机从"worklet 构建失败后"前移到"open() 开始时"。
  - 插件事件统一要求 `plugin:` 前缀，`PluginHost` 持有独立 `EventBus<PluginEventMap>`，与主库 EventBus 完全隔离。`RecorderController.on()` 按前缀路由，用户无需感知底层 bus 分离。
  - `frame` 事件降为内部事件，对外公开 `frame:async`（queueMicrotask 派发）。`hasAsyncFrameListeners` 标志在 `start()` 时缓存，热路径零开销。
- 踩坑：
  - `plugin-event-bus.spec.ts` 原先传 `EventBus<RecorderEventMap>` 给 `PluginEventBus`，重构后 bus 类型改为 `EventBus<PluginEventMap>`，测试需同步更新。
  - jsdom 环境下 `checkRecorderCapability()` 返回 `"unsupported"`，`recorder-controller.spec.ts` 中 `inputStrategy` 测试需允许 `undefined`（unsupported 时不写入）。
- 测试结果：
  - `capability-check.spec.ts` 新增 8 个用例，`recorder-controller.spec.ts` 新增 5 个用例，`plugin-host.spec.ts` 同步更新。
- 变更文件：

## PR4 — MediaRecorder 支持

- 目标：MediaRecorder（audio/webm;codecs=pcm）作为最高优先级采集路径，兼容不支持时自动降级。
- 完成：
  - `webm-pcm-extractor.ts` 移植 vendor WebM_Extract，改为提取全部声道（planar Float32Array[]），保持 scope 对象跨回调累积字节。Chrome v66 bitDepth=16+FLOAT 特殊处理内置。
  - `createMediaRecorderInputGraph()` 500ms 超时验证是核心——`isTypeSupported` 返回 true 但实际不产出数据的情况在真实设备上出现过（vendor 已踩坑记录在案）。
  - `inputNode` 用 `GainNode(gain=0)` 而非 `createMediaStreamSource()`。后者是 source node（inputs=0），`BrowserInputSession` 构造时 `sourceNode.connect(inputNode)` 会抛 `input index (0) exceeds number of inputs (0)`。GainNode 有 inputs，静音输出不影响实际录音。
  - 采样率不匹配（`webmSR !== audioContext.sampleRate`）时发出 warning 但继续投递，不强制降级。
  - `preferMediaRecorder: false` 允许一键跳过 MediaRecorder 路径，便于排查问题时回退到 AudioWorklet。
- 踩坑：
  - `BrowserInputSession` 与 `createMediaStreamSource` 的连接问题：MediaRecorder 路径下 `inputNode` 必须是有 inputs 的节点，不能复用 source node。
  - WebM EBML 解析：Segment 的 length 字段是"unknown size"（0x01ffffff...），`readBlock` 中 `len >= 0x7fffffff` 时不读取 payload，返回空数组——Segment 内容需继续逐元素解析，不能整块读取。
  - 多次调用 `webmExtract` 时 `scope.pos[0]` 需要在每次调用结束后正确保存，避免下次调用重复解析已处理的字节。
- 测试结果：
  - `webm-pcm-extractor.spec.ts` 新增 8 个用例（mono/stereo、流式累积、无效格式、bad 标记持久、多 SimpleBlock）。
  - `input-graph.spec.ts` 新增 5 个用例（preferMediaRecorder=false、无 stream、isTypeSupported=false、start 抛异常、500ms 超时）。
- 变更文件：
  - `src/input/webm-pcm-extractor.ts`（新增）
  - `src/input/input-graph.ts`
  - `src/input/browser-input-adapter.ts`
  - `src/input/capability-check.ts`
  - `src/types.ts`
  - `tests/unit/webm-pcm-extractor.spec.ts`（新增）
  - `tests/unit/input-graph.spec.ts`
